# Segmentation Theory -- GrainSight

This document describes the grain segmentation algorithms implemented in
`app/simulation/segmentation.py`. GrainSight ships three complementary
segmentation methods, each tuned to a different class of imaging
condition.

---

## 1. Problem Statement

Given an RGB-D image of a granular bed, the goal is to produce a
per-pixel integer label image where each positive label identifies a
single grain footprint and `0` denotes background (inter-grain gaps or
sub-threshold regions).

Formally, given:

- depth map `z : Omega -> R+` on the image domain `Omega subset Z^2`,
- optional colour image `I : Omega -> [0, 255]^3`,

we seek a labelling `L : Omega -> {0, 1, ..., K}` that partitions the
foreground into `K` connected grain regions.

The principal challenge is **touching grains**: neighbouring grains
separated by a thin valley must be split into distinct labels rather
than merged into one blob.

---

## 2. Method 1 -- Marker-Based Watershed (default)

Function: `segment_grains_watershed(depth, rgb=None, ...)`.

### 2.1 Pipeline

1. **Smooth** the depth map with a Gaussian kernel:
   `z_s = G_sigma * z`, suppressing sensor noise while preserving edges
   at the scale of interest.

2. **Compute gradient magnitude** using Sobel operators:

   ```
   |nabla z_s| = sqrt((dz_s/dx)^2 + (dz_s/dy)^2)
   ```

   Grain boundaries (depth discontinuities) produce high gradient
   values.

3. **Foreground mask**: `F = { p : z_s(p) > tau }` with
   `tau = 0.1 * max(z_s)` by default, excluding background from the
   flood domain.

4. **Marker seeds**: local maxima of `z_s` (grain peaks) found via
   `peak_local_max` with a minimum separation `min_distance` and a
   relative threshold `peak_threshold_rel`. Each maximum receives a
   unique integer label.

5. **Watershed transform**: flooding of the topographic surface
   `|nabla z_s|` from the markers, restricted to `F`. Watershed
   lines lie on the ridges of the gradient field -- i.e. at the true
   grain boundaries.

6. **Post-processing**: regions smaller than `min_grain_size` pixels
   are merged into their largest neighbour, then labels are
   re-numbered to be contiguous.

### 2.2 When to use

- Industrial conveyor imaging with clean, high-contrast depth.
- Grains separated by clear depth valleys (typical of crushed rock).
- Uniform lighting -- colour gradient adds little information.

---

## 3. Method 2 -- Depth Edges (lightweight)

Function: `segment_grains_depth_edges(depth, edge_threshold=2.0, ...)`.

A simpler heuristic: threshold the gradient magnitude to declare
pixels as **boundary** or **foreground**, then label connected
components of the foreground. Fast, no marker detection, but tends
to oversegment highly-textured depth.

Best for quick previews or as a baseline.

---

## 4. Method 3 -- Combined RGB-D Fusion

Function: `segment_grains_rgbd(depth, rgb, depth_weight, color_weight, ...)`.

### 4.1 Motivation

Depth alone fails when grains are nearly the same height but
visually distinct (different mineralogy, surface texture, colour).
Colour alone fails under shadows, specular highlights, or uniform
lithology. Fusing both channels recovers boundaries that are
visible in either modality.

### 4.2 Fused gradient

Given normalised gradients `|nabla z|_n` and `|nabla I|_n` in `[0, 1]`:

```
|nabla F| = w_d * |nabla z|_n + w_c * |nabla I|_n,   w_d + w_c = 1
```

The frontend exposes `Depth weight` and `Color weight` sliders that
stay linked so the sum remains 1.

### 4.3 Markers & watershed

Markers are still taken from peaks of the smoothed depth map (the
depth channel retains the most reliable "one peak per grain"
signal). The watershed is then run on the fused gradient `|nabla F|`,
so flood ridges follow **either** depth valleys or colour
boundaries, whichever is locally stronger.

### 4.4 Recommended weights

| Scene                        | `depth_weight` | `color_weight` |
|------------------------------|----------------|-----------------|
| Uniform lighting conveyor    | 0.8 - 0.9      | 0.2 - 0.1       |
| Mixed lithology bench muck   | 0.6            | 0.4             |
| Textured surface, low relief | 0.3 - 0.4      | 0.7 - 0.6       |
| Pure colour segmentation     | 0.0            | 1.0             |

Defaults (`w_d = 0.6`, `w_c = 0.4`) work well on the synthetic beds
generated by `grain_generator.py`.

### 4.5 Limitations

- Requires both RGB and depth channels to be registered (same
  resolution, no parallax). The generator produces perfectly
  registered pairs; real sensors need a prior alignment step.
- Markers still come from depth, so pure colour-only separations
  (two adjacent grains at identical height but different colour)
  will share a marker and merge. A future extension could seed
  markers from combined depth-plus-colour distance transforms.

---

## 5. Parameters Summary

| Parameter              | Effect                                                      |
|------------------------|-------------------------------------------------------------|
| `smooth_sigma`         | Higher = more noise suppression, rounder boundaries         |
| `min_distance`         | Minimum pixel gap between grain peaks (prevents duplicates) |
| `peak_threshold_rel`   | Relative peak intensity threshold (0-1)                     |
| `min_grain_size`       | Merge regions smaller than this into neighbours             |
| `depth_edge_threshold` | Method 2 only: gradient cutoff for boundary pixels          |
| `depth_weight`         | Method 3 only: fusion weight for depth gradient             |
| `color_weight`         | Method 3 only: fusion weight for colour gradient            |
